# 添加照片到照片墙处理流程 Skill

这是 `lccandsr.github.io` 情侣照片墙项目添加新照片的标准化处理流程。

---

## 完整步骤

### 1. 接收图片并识别
- 用户发送图片后，先识别图片内容
- 按照要求格式回复：**"这么棒的宝宝，[图片描述]。这张照片超有氛围感呀～请问需要添加到照片墙中吗？"**
- 等待用户确认，用户说不加入就结束，加入继续下一步

### 2. 保存图片到正确目录
- 获取当前年月，判断属于哪个年份和季节：
  - 年份：就是当前年份 `YYYY`
  - 季节：
    - 3-5月：spring (春天)
    - 6-8月：summer (夏天)
    - 9-11月：autumn (秋天)
    - 12-2月：winter (冬天)
- 目标路径：`new-images/{年份}/{季节}/`
- 如果目录不存在，创建目录 `mkdir -p 目标路径`
- 图片命名：找当前目录下最大序号+1，比如已有 47.jpg，下一张就是 48.jpg
- 复制图片到目标路径：`cp 原图片 目标路径/{序号}.jpg`
- （可选：如果图片太大，可以适当压缩）

### 3. 更新配置文件
- 编辑 `images.json`，在对应年份季节的数组中添加新图片文件名
- 同步编辑 `new-index.html` 中的默认 `imageData`，保持和 json 一致，避免缓存问题

### 4. 同步持久化数据
- 运行同步脚本更新访问次数备份：`node sync-counter.js`
- 如果有新增弹幕，同步 `danmaku.json`：从 localStorage 获取最新弹幕，覆盖保存到项目文件

### 5. 推送到 GitHub
按照以下顺序执行命令：
```bash
cd /Users/bytedance/Documents/code/my-code/lccandsr.github.io
git add .
git commit -m "Add new photo: [年份]/[季节]/[文件名] - [简短描述]"
git push
```

### 6. 反馈结果
- 如果推送成功，告诉用户：✅ 推送完成！过一会刷新页面就能看到最新照片了～
- 如果推送失败，把错误信息展示给用户，询问如何处理

---

## 补充说明

- **图片序号规则**：同一个季节文件夹中，按顺序递增命名，不跳号不重复
- **配置一致性**：必须同时更新 `images.json` 和 `new-index.html` 中的默认数据，避免缓存导致不显示
- **持久化**：每次操作完都要同步计数器，保证本地 `counter.json` 是最新备份
- **提交信息**：commit 信息要清晰说明添加了什么照片，方便后续回溯

## 示例

用户发了一张2026年3月的照片：
- 年份：2026，季节：spring
- 当前文件夹已有 46.jpg, 47.jpg, 48.jpg → 命名 49.jpg
- 路径：`new-images/2026/spring/49.jpg`
- commit 信息：`Add new photo: 2026/spring/49.jpg - 宝宝甜酷对镜拍`
